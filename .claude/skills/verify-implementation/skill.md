# verify-implementation

実装・修正が完了したら、以下のステップで動作確認を行い、結果を報告する。

## Step 1: 構文チェック

変更した `.sh` ファイルを対象に構文チェックを実行する:

```
bash -n <changed-file>.sh
```

エラーがあれば修正してから次のステップへ進む。

## Step 2: 実際に実行して動作確認

```
bash tmux-sidebar.sh
```

以下を目視で確認する:

- サイドバーペインが正しく開くか
- サイドバーのサイズ・位置が意図通りか
- トグル動作（開く → 閉じる → 開く）が正しく機能するか

## Step 3: エッジケースの確認

以下のシナリオを順に試す:

1. **二重実行**: サイドバーが既に開いている状態で再度 `bash tmux-sidebar.sh` を実行する
2. **tmux 外での実行**: tmux セッション外（通常ターミナル）で実行し、適切なエラーメッセージが出るか確認する

## Step 4: 既存レイアウトへの影響確認

- 既存 tmux ウィンドウのレイアウトが壊れていないか確認する
- キーバインドの競合がないか確認する

## Step 5: 結果の報告

確認完了後、以下の形式で報告する:

```
## 動作確認結果

### 確認済みシナリオ
- [ ] 構文チェック: bash -n の出力
- [ ] 通常起動: サイドバー開閉の動作
- [ ] 二重実行: 挙動
- [ ] tmux 外実行: エラーメッセージ

### 未確認シナリオ（ある場合のみ）
- 理由とともに記載

### エラー・懸念事項（ある場合のみ）
- 内容を記載
```
